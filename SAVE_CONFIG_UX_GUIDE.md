# 系统配置保存体验优化说明

## 🎯 优化目标

解决用户反馈的问题：
1. ❌ 修改地图提供商后，前端地图没有更新
2. ❌ 配置显示"已修改"但不知道如何保存
3. ❌ 没有明确的保存/取消按钮

## ✅ 优化后的体验

### 1. **双重确认机制**

#### 顶部警告提示
```
┌─────────────────────────────────────────────┐
│ ⚠️  您有 2 项未保存的更改                    │
│ 请点击底部的"保存更改"按钮以应用配置，       │
│ 或点击"取消"放弃修改。                       │
└─────────────────────────────────────────────┘
```

#### 底部固定保存栏
```
┌────────────────────────────────────────────────────────┐
│ 🔵 您有 2 项未保存的更改                               │
│ • 地图提供商, 系统时区                                 │
│                                    [取消] [保存更改]   │
└────────────────────────────────────────────────────────┘
```

### 2. **实时状态反馈**

| 操作 | 页面反馈 |
|------|---------|
| 修改配置 | ✅ 配置项显示"已修改"蓝色标签 |
| 继续修改 | ✅ 顶部警告 + 底部保存栏更新数量 |
| 保存成功 | ✅ 绿色成功提示 + 清除未保存状态 |
| 地图配置 | ✅ 特殊提示："3秒后自动刷新页面" |

### 3. **地图配置自动刷新**

当您修改地图相关配置（`map.provider` 或 `map.api_key`）并保存后：

```typescript
// 系统自动检测
if (修改了地图配置) {
  显示提示("地图配置已更新，3秒后自动刷新页面...");
  等待3秒;
  刷新页面(); // 新地图配置立即生效
}
```

**支持的地图提供商：**
- `carto` - CartoDB (默认，免费)
- `openstreetmap` - OpenStreetMap (免费)
- `mapbox` - Mapbox (需要 API key)

## 🔄 完整交互流程

### 场景 1: 修改普通配置

```
1. 用户修改"心跳间隔"从 30000 → 60000
   ↓
2. 配置项显示"已修改"蓝色标签
   ↓
3. 顶部出现琥珀色警告提示
   ↓
4. 底部出现蓝色固定保存栏
   ↓
5. 用户点击"保存更改"
   ↓
6. 显示绿色成功提示
   ↓
7. 警告和保存栏消失
   ↓
8. 配置立即生效（无需刷新）
```

### 场景 2: 修改地图配置

```
1. 用户修改"地图提供商" carto → openstreetmap
   ↓
2. 配置项显示"已修改"蓝色标签
   ↓
3. 顶部出现警告 + 底部出现保存栏
   ↓
4. 用户点击"保存更改"
   ↓
5. 显示特殊提示:
   "成功更新了 1 个配置项。
    地图配置已更新，3秒后自动刷新页面..."
   ↓
6. 3 秒倒计时
   ↓
7. 页面自动刷新
   ↓
8. 地图组件使用新的 openstreetmap 提供商
```

### 场景 3: 取消修改

```
1. 用户修改了多个配置
   ↓
2. 底部保存栏显示"您有 5 项未保存的更改"
   ↓
3. 用户点击"取消"按钮
   ↓
4. 弹出确认对话框:
   "确定要放弃所有未保存的更改吗？"
   ↓
5a. 点击"确定" → 所有更改被撤销
5b. 点击"取消" → 继续编辑
```

### 场景 4: 单个配置撤销

```
1. 用户修改了 3 个配置
   ↓
2. 每个配置项右下角都有"撤销"按钮
   ↓
3. 用户点击其中一个配置的"撤销"
   ↓
4. 该配置恢复原值，"已修改"标签消失
   ↓
5. 保存栏数量更新: "您有 2 项未保存的更改"
   ↓
6. 如果撤销所有更改 → 保存栏消失
```

## 🎯 关键功能

### 1. 智能地图配置检测

```typescript
// 保存时检查是否修改了地图配置
const hasMapConfigChange = configsToUpdate.some(
  c => c.key.startsWith('map.')
);

if (hasMapConfigChange) {
  // 特殊处理：刷新页面以重新加载地图组件
  setTimeout(() => window.location.reload(), 3000);
}
```

### 2. 未保存状态管理

```typescript
// 使用 Map 跟踪所有未保存的更改
const [changedConfigs, setChangedConfigs] = useState<Map<string, string>>(
  new Map()
);

// 修改配置时
handleConfigChange(key, newValue) {
  if (newValue === originalValue) {
    changedConfigs.delete(key); // 恢复原值时移除
  } else {
    changedConfigs.set(key, newValue); // 添加到未保存列表
  }
}
```

```typescript
</div>
```

## 📝 使用建议

1. **修改配置后记得保存**
   - 看到底部蓝色保存栏时，记得点击"保存更改"

2. **地图配置需要刷新**
   - 修改地图提供商后，保存会自动刷新页面
   - 请耐心等待 3 秒倒计时

3. **批量修改更高效**
   - 可以一次性修改多个配置
   - 统一点击"保存更改"一次性保存

4. **撤销单个更改**
   - 每个配置项都有独立的"撤销"按钮
   - 不影响其他未保存的更改

5. **取消所有更改需确认**
   - 底部"取消"按钮会二次确认
   - 避免误操作丢失修改
