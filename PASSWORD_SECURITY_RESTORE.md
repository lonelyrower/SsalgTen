# 🔐 账户安全功能恢复说明

## 问题描述

在之前的UI优化中，密码修改功能从系统配置页面被移除，导致用户无法修改密码。这是一个安全隐患，因为所有用户默认使用相同密码。

## 解决方案

### ✅ **在系统配置页面顶部添加醒目的"账户安全"卡片**

**设计特点**:

1. **高优先级位置**: 放在页面顶部，搜索栏上方
2. **醒目的视觉设计**:
   - 渐变背景 (`from-orange-50 to-red-50`)
   - 橙红色边框 (`border-orange-200`)
   - Lock 图标 + 渐变背景圆角容器
   - "重要"徽章标识

3. **清晰的操作引导**:
   - 主标题: "账户安全"
   - 说明文字: "修改您的登录密码，保护账户安全"
   - 提示信息: "建议定期更换密码，密码长度至少6个字符"
   - 醒目的"修改密码"按钮 (渐变效果 `from-orange-600 to-red-600`)

4. **使用已有的 ChangePasswordModal 组件**:
   - 模态框形式，不干扰配置页面布局
   - 完整的表单验证
   - 密码可见性切换
   - 成功/失败通知

## 代码变更

### 文件: `frontend/src/components/admin/SystemSettings.tsx`

**新增导入**:

**新增状态**:

**新增卡片** (位置: 头部和搜索栏之间):
```tsx
<Card className="p-6 bg-gradient-to-br from-orange-50 to-red-50 ...">
  <div className="flex items-start justify-between">

      <Lock className="h-6 w-6 text-white" />
    </div>
    
    {/* 标题和说明 */}
    <div>
      <h3>账户安全 <span>重要</span></h3>
      <p>修改您的登录密码，保护账户安全</p>
      <AlertCircle /> 建议定期更换密码，密码长度至少6个字符
    </div>
    
    {/* 修改密码按钮 */}
    <Button onClick={() => setShowChangePasswordModal(true)}>
      <Key /> 修改密码
    </Button>
  </div>
</Card>
```

**新增模态框**:
```tsx
<ChangePasswordModal
  isOpen={showChangePasswordModal}
  onClose={() => setShowChangePasswordModal(false)}
/>
```

## 密码修改流程

### **用户操作步骤**:
1. 进入"系统管理" → "系统配置"标签页
2. 看到顶部橙红色"账户安全"卡片
3. 点击"修改密码"按钮
4. 在模态框中填写:
   - 当前密码
   - 新密码
   - 确认新密码
5. 点击"修改密码"提交
6. 成功后收到通知，模态框自动关闭

### **表单验证**:
- ✅ 当前密码不能为空
- ✅ 新密码长度至少6个字符
- ✅ 新密码不能与当前密码相同
- ✅ 两次新密码输入必须一致

---

## 安全性保障

### **密码策略**:
- 最小长度: 6个字符
- 不能与当前密码相同
- 建议定期更换

### **API安全**:
- 使用 `apiService.changePassword(currentPassword, newPassword)`
- 后端验证当前密码正确性
- 密码传输加密（HTTPS）
- 成功后返回通知
