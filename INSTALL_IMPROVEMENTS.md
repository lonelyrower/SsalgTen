# SsalgTen 安装脚本改进说明

## 问题描述

根据实际部署测试，发现以下问题：

### 1. 低内存VPS数据库启动超时
**现象：**
- 在1G内存VPS上：数据库启动超时（[ERROR] 数据库启动超时）
- 在4G内存VPS上：长时间卡在"等待数据库启动"阶段

**原因：**
- 原等待时间设置：30次 × 2秒 = 60秒
- PostgreSQL在低内存环境下启动需要更长时间
- 没有提供足够的等待时间和进度提示

### 2. 部署成功提示信息不完整
**现象：**
- 部署成功后只显示访问地址
- 缺少默认账号密码信息
- 用户不知道如何登录和使用

## 修复方案

### 1. 数据库启动超时修复

#### 改进点：
- ✅ 增加等待时间：从30次增加到60次
- ✅ 增加检查间隔：从2秒增加到3秒
- ✅ 总等待时间：从60秒增加到180秒（3分钟）
- ✅ 添加详细进度提示
- ✅ 添加容器状态检查
- ✅ 失败时显示完整日志和排查建议

#### 新增功能：
```bash
# 1. 等待提示更友好
log_info "[INFO] 数据库启动可能需要1-3分钟，请耐心等待..."
log_info "[INFO] 低内存VPS (1G-2G) 可能需要更长时间，这是正常现象"

# 2. 实时显示已等待时间
等待数据库启动... (15/60) - 已等待 45秒

# 3. 检查容器状态
if ! docker_compose -f $compose_file ps database | grep -q "Up"; then
    log_error "数据库容器已停止，检查日志..."
fi

# 4. 超时后显示详细错误信息
log_error "数据库启动超时 (等待了 180秒)"
echo "=== 数据库日志 ==="
docker_compose -f $compose_file logs --tail=100 database
echo "可能的原因:"
echo "  1. 内存不足 (建议至少2G内存)"
echo "  2. 磁盘空间不足"
echo "  3. Docker资源限制"
```

## 相关问题

### 如何进一步优化？

如果仍然遇到数据库启动问题：

1. **增加swap空间**
```bash
# 创建2G swap
sudo fallocate -l 2G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

2. **优化Docker资源**
```bash
# 清理Docker缓存
docker system prune -af
```

3. **使用镜像安装**
```bash
# 镜像安装避免构建过程，减少资源消耗
curl -fsSL https://raw.githubusercontent.com/lonelyrower/SsalgTen/main/scripts/ssalgten.sh | bash -s -- deploy --image
```
