# 🏗️ SsalgTen 统一脚本架构方案

## 📋 架构概述

通过整合现有的3个脚本，构建统一、高效的部署和管理体系：

```
🔧 整合前 (3个脚本)           🎯 整合后 (2个核心脚本)
┌─────────────────────┐      ┌──────────────────────────┐
│ deploy-production.sh│ ───► │  deploy-production.sh    │
│ (初始部署)           │      │  • 系统环境配置          │
└─────────────────────┘      │  • 镜像/源码双模式       │
                            │  • SSL + 域名配置        │
┌─────────────────────┐      └──────────────────────────┘
│ ssalgten.sh         │ ───► ┌──────────────────────────┐
│ (源码管理)           │      │  ssalgten.sh (统一管理)   │
└─────────────────────┘      │  • 源码模式 (现有)       │
                            │  • 镜像模式 (新增)       │
┌─────────────────────┐      │  • 智能模式检测          │
│ ghcr-deploy.sh      │ ──┐  │  • 数据备份/监控        │
│ (镜像管理)           │   └► │  • 模式切换功能          │
└─────────────────────┘      └──────────────────────────┘
```

- **从3个脚本 → 2个脚本**：减少用户学习成本

- **智能检测**：自动识别当前部署模式

### **2. 部署效率提升**

```
│ 镜像模式    │     -      │  2-4分钟   │

日常更新时间:

│ 镜像模式    │  1-2分钟    │  1-2分钟    │
```

### **3. 用户体验优化**
- **渐进式迁移**：现有用户无缝升级
- **灵活选择**：初始部署时可选模式
- **随时切换**：运行时可切换部署模式


## 🏗️ 技术架构

### **1. 模式检测系统**

```bash
# 部署模式标记文件
/opt/ssalgten/.deployment_mode
│
├── "image"  ── 镜像模式
└── "source" ── 源码模式 (默认)

# 智能检测函数
detect_deployment_mode() {
    if [[ -f "/opt/ssalgten/.deployment_mode" ]]; then
        cat "/opt/ssalgten/.deployment_mode"
    else
        echo "source"  # 向后兼容
    fi
}
```

### **2. 配置管理系统**

```bash
# 配置文件分离
/opt/ssalgten/
├── .env              # 源码模式配置 (现有)
├── .env.image        # 镜像模式配置 (新增)
├── .deployment_mode  # 模式标记 (新增)
└── docker-compose.ghcr.yml  # 镜像compose文件
```

### **3. 命令统一接口**

```bash
# 统一命令格式
./ssalgten.sh <action> [--mode] [options]

# 示例:
./ssalgten.sh start           # 自动检测模式启动
./ssalgten.sh start --image   # 强制镜像模式
./ssalgten.sh update --image  # 镜像模式更新
./ssalgten.sh switch image    # 切换到镜像模式
```

| **网络要求** | 📥 拉取镜像 | 📥 拉取源码+依赖 |

## 📝 关键实现要点

### **1. 向后兼容性保证**

```bash
# 现有命令完全保持不变
./ssalgten.sh start     # 继续有效
./ssalgten.sh update    # 继续有效
./ssalgten.sh backup    # 继续有效

# 自动检测模式，无需用户关心
```

### **2. 配置迁移机制**

```bash
# 自动从现有配置迁移到镜像配置
migrate_to_image_config() {
    if [[ -f ".env" && ! -f ".env.image" ]]; then
        # 自动迁移关键配置
        source .env
        create_image_config_from_source
    fi
}
```

### **3. 智能错误处理**

```bash
# 镜像模式失败时的自动降级
start_with_fallback() {
    if ! start_image_mode; then
        log_warning "镜像模式启动失败，尝试源码模式..."
        start_source_mode
    fi
}
```
